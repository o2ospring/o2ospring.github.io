前言

Windows Subsystem for Linux (WSL) 是微软开发的 Windows 系统组件，能让用户在 Windows 上直接运行完整的 Linux 发行版，无需虚拟机或双系统。

WSL 简介

1、核心特性：
  1. 原生集成：与 Windows 系统深度融合，可直接访问 Windows 文件系统（如 C 盘、D 盘），也支持 Linux 与 Windows 程序相互调用。
  2. 轻量高效：无需单独分配硬件资源，启动速度快，资源占用远低于传统虚拟机。
  3. 完整 Linux 体验：支持主流 Linux 发行版（如 Ubuntu、Debian、Fedora），包含完整的 Linux 命令行工具、包管理器和软件生态。
  4. 双系统互通：可在 WSL 中使用 Windows 的环境变量，也能在 Windows 命令行（CMD/PowerShell）中调用 Linux 命令。
2、版本差异：
版本 核心特点 适用场景
WSL 1 基于翻译层实现，启动更快 简单命令行操作、轻量开发
WSL 2 基于 Hyper-V 虚拟机技术，支持完整 Linux 内核 复杂开发环境、容器（Docker）、图形界面应用
3、适用场景：
  • 开发者在 Windows 上搭建 Linux 开发环境，无需切换系统。
  • 运行 Linux 专属工具、脚本或服务（如 Shell 脚本、Node.js、Python 后端）。
  • 学习 Linux 系统操作，无需单独安装 Linux 系统。
  • 适配跨平台项目，确保开发环境与生产环境（Linux 服务器）一致。
4、注意事项：
  • 优势：兼顾 Windows 的易用性和 Linux 的生态，开发效率高，配置简单。
  • 注意事项：WSL 2 需开启 CPU 虚拟化功能（需在 BIOS 中设置），部分对硬件要求极高的 Linux 应用（如 3D 渲染）可能兼容性有限。
